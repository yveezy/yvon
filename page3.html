Mode CLI

Pour utiliser notre application en mode terminal, nous avons créé la class Root.java

Elle interprète les commandes saisies par l'utilisateur pour déclencher des analyses et des corrections spécifiques sur les fichiers Python. Lorsqu’elle est appelée, cette classe prend en compte différentes options telles que -h pour afficher l’aide, -d pour analyser un répertoire, ou -f pour examiner un fichier spécifique. Elle propose aussi à l’utilisateur d’utiliser des options qui permet de vérifier et d’améliorer le fichier python.

Notre appli propose 5 manières d’utiliser les options :

--type pour vérifier la présence des annotations types.

--head pour vérifier la présence du shebang

--pydoc pour vérifier la présence des commentaires pydoc.

--sbutf8 pour ajouter le shebang.

--comment pour ajouter les commentaire pydoc.

Mode GUI

Le GUI offre une interface pour manipuler les fichiers Python. Il propose des boutons pour sélectionner des répertoires, lister les fichiers Python, analyser un fichier ou un répertoire, ajouter des éléments tels que les shebangs ou les commentaires, et générer des statistiques. Il présente une fenêtre avec des boutons d'action, un champ texte et une zone de résultat. Chaque bouton déclenche des actions distinctes : le bouton "Analyser Fichier" ouvre un explorateur pour choisir un fichier Python à analyser et affiche le résultat. "Ajouter Shebang" insère un shebang dans un fichier sélectionné. "Lister Fichiers Python" présente la liste des fichiers Python dans un répertoire choisi. "Ajouter Commentaire" vérifie l'existence de commentaires et les ajoute si nécessaire. "Générer Statistique" produit des statistiques pour le répertoire sélectionné. Ces actions, définies dans des classes internes, facilitent l'interaction avec les fichiers Python, offrant une expérience conviviale pour analyser, modifier et visualiser ces fichiers de manière interactive.
