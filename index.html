Introduction

Notre projet consiste à concevoir une application destinée à améliorer la qualité des codes Python. Cette application a pour objectif de détecter les annotations de type manquantes, ce qui permet d'ajouter des commentaires de documentation standardisés, facilitant ainsi la génération de documentation ultérieure. De plus, elle offre la possibilité de répertorier les fichiers Python dans un dossier spécifique, de détecter l'absence de commentaires ou d'annotations, et d'extraire des statistiques sur la qualité du code.

Pour l'utilisation de notre application, nous proposons deux modes d'utilisation prévus. Tout d'abord, il y a le mode en ligne de commande (CLI) qui permet une utilisation directe et efficace, idéale pour les tâches automatisées et la gestion des fichiers en lot. Ensuite, nous avons développé une interface graphique (GUI) qui offre une expérience visuelle conviviale pour explorer et modifier les fichiers de manière interactive.

Cette approche permet aux utilisateurs de travailler de manière plus intuitive tout en maintenant la qualité de leur code Python.

En somme, notre projet vise à fournir une solution complète pour améliorer la qualité du code Python en détectant les annotations de type manquantes, en facilitant la création de commentaires de documentation standardisés, en offrant des statistiques sur la qualité du code, et en proposant deux modes d'utilisation flexibles : en ligne de commande (CLI) et en interface graphique (GUI).
